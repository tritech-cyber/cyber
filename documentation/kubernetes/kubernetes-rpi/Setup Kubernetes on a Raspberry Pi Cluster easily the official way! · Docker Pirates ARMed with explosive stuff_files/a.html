                        I just solved my problem with the following:

                        Since kubeadm 1.6 the security for netwerking is more secure. So for Flannel getting to work you need to do the following:

                        1. install Kubernetes
                        2. for Flannel; first you need to do: kubeadm apply -f https://raw.githubuserconte...
                        3. curl -sSL https://raw.githubuserconte... | sed "s/amd64/arm/g" | kubectl create -f -

                        Since Docker 1.13 the ipatables rules are changed, for that you need to do:

                        1. sudo iptables -A FORWARD -i cni0 -j ACCEPT
                        2. sudo iptables -A FORWARD -o cni0 -j ACCEPT

                        Then it should work.

                                                        sudo iptables -A FORWARD -i cni0 -j ACCEPT
                                                        sudo iptables -A FORWARD -o cni0 -j ACCEPT


                                                        on each node and the commands of the tutorial are working seamlessly again.

                                                              I was having this same flannel issues (using Rasbian). I first downgraded docker to 1.12.6 and it did resolve the problem.                                                                                           • Tomasz Jdrzejewski Debi Naik 3 months ago

                                                                                            Hi Debi! I managed to run a private Docker registry on a RPI Kubernetes cluster. What you need to do is to follow this guide: https://github.com/kubernet...

                                                                                            You deploy kube-registry on one of your nodes, and then also kube-registry-proxy that would expose it as localhost:5000 on every node. You must use ARM images - I used: kubernetesonarm/kube-registry-proxy-arm:0.4 and budry/registry-arm:latest

                                                                                            The next thing is that `hostPort` attribute does is ignored in CNI networks, there is an issue for that: https://github.com/kubernet...

                                                                                            I hope they will finish it soon. Meanwhile, you can apply the following workaround:
                                                                                            1. start kube-registry on port 5001
                                                                                            2. for kube-registry-proxy DaemonSet, add the following attribute to the spec: "hostNetwork: true" - this will make all the node interfaces visible for the pod,
                                                                                            3. still in the DaemonSet:
                                                                                            - set both containerPort and hostPort to 5000,
                                                                                            - set REGISTRY_PORT to 5001

                                                                                            Now, if your cluster DNS service is working correctly, this should work. Note that REGISTRY_HOST environment variable accepts a hostname, and it won't work with IP. If you see in the logs that kube-registry-proxy is waiting for kube-registry to go online, check your kube-dns setup. Final test: `curl http://localhost:5000` shall return an empty result on all nodes.

                                                                                            The workaround has some drawbacks - the registry is visible in the entire network, but I'm using my cluster only for experiments and this is acceptable for me.

                                                                                                  I've been struggling with kubernetes v1.6.2 and Raspberry Pi, and when I realized that this version is using RBAC, all those things start making sense...

                                                                                                  To build a kubernetes cluster v1.6.2 over a Raspberry you should create the appropiate RBAC Roles before creating the flannel configmap and the daemonset.

                                                                                                  The steps should look like this:

                                                                                                  kubeadm init
                                                                                                  --pod-network-cidr=10.244.0.0/16

                                                                                                  cp /etc/kubernetes/admin.conf $HOME/
                                                                                                  chown $(id -u):$(id -g) $HOME/admin.conf
                                                                                                  export KUBECONFIG=$HOME/admin.conf

                                                                                                  kubectl create -f
                                                                                                  https://raw.githubuserconte...

                                                                                                  curl -sSL
                                                                                                  https://raw.githubuserconte...
                                                                                                  | sed "s/amd64/arm/g" | kubectl create -f -

                                                                                                  After these steps you will get the following:

                                                                                                  kubectl -n kube-system get pod
                                                                                                  NAME READY STATUS RESTARTS AGE
                                                                                                  etcd-raspberrypi 1/1 Running 0 2m
