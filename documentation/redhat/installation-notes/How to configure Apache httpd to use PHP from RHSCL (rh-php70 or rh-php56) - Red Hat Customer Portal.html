How to configure Apache httpd to use PHP from RHSCL (rh-php70 or rh-php56)

Solution Verified - Updated June 6 2017 at 6:24 PM -
English
  • English
  • 日本語

Environment

  • Red Hat Enterprise Linux 7
  • Red Hat Enterprise Linux 6
  • httpd or http24 with one of:
    • rh-php70-php
    • rh-php56-php
    • php55-php
    • php54-php

Issue

  • How can we connect mod_php7 (libphp7) from Red Hat Software Collections (RHSCL) rh-php70-php to httpd?
  • How can we connect mod_php5 (libphp5) from RHSCL rh-php56-php to httpd?
  • How can we connect mod_php5 (libphp5) from RHSCL php55-php to httpd?
  • How can we connect mod_php5 (libphp5) from RHSCL php54-php to httpd?
  • I would like to know what I need to install and how to configure it so that httpd24 is being used with php56 (or greater) and my .php files are being processed by the Apache Web Server.
  • How to install and configure Apache httpd and PHP 5.6 from RHSCL?

Resolution

Overview

  • Warning: the php54 and php55 collections were retired in October 2016 and new deployments are not recommended
    Ref: Red Hat Software Collections Product Life Cycle

  • rh-php70-php, rh-php56-php, and php55-php are built against httpd24 from Red Hat Software Collections (RHSCL)
    They will NOT work with the default httpd packages shipped in RHEL 6 or RHEL 7

    ~]# rpm -ql rh-php70-php | grep -- -php7.so
/opt/rh/httpd24/root/usr/lib64/httpd/modules/librh-php70-php7.so
~]# rpm -ql php55-php | grep -- -php5.so
/opt/rh/httpd24/root/usr/lib64/httpd/modules/libphp55-php5.so

  • php54-php is built against httpd from standard RHEL 6 and RHEL 7 channels
    It will NOT work with the httpd24 packages shipped in RHSCL

    ~]# rpm -ql php54-php | grep -- -php5.so
/usr/lib64/httpd/modules/libphp54-php5.so

Consider php-fpm instead of mod_php

  • Why not to use mod_php ...

    • The mod_php approach is the slowest possible way to serve php content in Apache httpd; it also is the least efficient, using the most memory
    • mod_php is not thread-safe, preventing use of modern threaded MPMs like worker and event
    • Ref: Running PHP on Apache httpd

  • Use php-fpm

    • All 4 collections (rh-php70, rh-php56, php55, & php54) include the separate php-fpm service which can serve PHP content back to Apache httpd
    • This option should be seriously considered as it provides a faster, more scalable setup that is thread-safe
    • Ref: Apache with various PHP versions, using SCL

Setup rh-php70 or rh-php56

  1. Ensure system has RHSCL repo/channel enabled
    How to use Red Hat Software Collections (RHSCL) or Red Hat Developer Toolset (DTS)

  2. Install any required httpd24 packages
    Example:

    Raw
    yum install httpd24 httpd24-mod_ssl

  3. Install php70 or php56 packages

    Raw
    yum install rh-php70 rh-php70-php

    Or:

    Raw
    yum install rh-php56 rh-php56-php

  4. Disable/stop old httpd service if it's installed
    (Note that it is possible to have both httpd and httpd24-httpd services running at the same time if the default Listen options are changed to avoid conflicts)

    Raw
    service httpd stop; chkconfig httpd off

    (The service & chkconfig commands work in both RHEL 6 and RHEL 7)

  5. Enable/start the new httpd24-httpd service

    Raw
    service httpd24-httpd start; chkconfig httpd24-httpd on

  6. Understand that the default httpd24 DocumentRoot resides under /opt

    ~]# /etc/cron.daily/mlocate 
~]# locate var/www/html
/opt/rh/httpd24/root/var/www/html
/var/www/html

  7. Test with a phpinfo() page
    Example, using the default DocumentRoot:

    ~]# echo '<?php phpinfo(); ?>' >/opt/rh/httpd24/root/var/www/html/phpinfo.php
~]# rpm -q elinks >/dev/null || yum install elinks
~]# elinks -dump http://localhost/phpinfo.php | head -15
   [1]PHP logo

   PHP Version 5.6.5

   System            Linux r72.example.com 3.10.0-327.el7.x86_64 #1 SMP Thu
                     Oct 29 17:29:29 EDT 2015 x86_64
   Build Date        Jul 25 2016 04:07:12
   Server API        Apache 2.0 Handler
   Virtual Directory disabled
   Support
   Configuration
   File (php.ini)    /etc/opt/rh/rh-php56
   Path
   Loaded
   Configuration     /etc/opt/rh/rh-php56/php.ini

Note
The main configuration file for php56 is /opt/rh/httpd24/root/etc/httpd/conf.d/rh-php56-php.conf and this module will load from /opt/rh/httpd24/root/etc/httpd/conf.modules.d/10-rh-php56-php.conf file.
